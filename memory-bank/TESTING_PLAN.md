# Укрупненный план по тестированию приложения

Этот документ описывает высокоуровневую стратегию тестирования для обеспечения качества и стабильности приложения.

## 1. Области тестирования

Тестирование будет разделено на несколько ключевых модулей:

-   **Модульное тестирование (Unit Tests):** Проверка отдельных функций, утилит и компонентов в изоляции.
-   **Интеграционное тестирование (Integration Tests):** Проверка взаимодействия между несколькими компонентами или модулями (например, форма + API).
-   **End-to-End (E2E) тестирование:** Проверка полных пользовательских сценариев в окружении, приближенном к реальному.

## 2. Приоритетные модули для покрытия тестами

1.  **Аутентификация и авторизация:**
    -   Логин / выход.
    -   Защита роутов в зависимости от роли пользователя (Admin, User, Guest).
    -   Доступ к конфиденциальным документам.

2.  **Управление документами (CRUD):**
    -   Загрузка, просмотр, редактирование и удаление документов.
    -   Работа с категориями и метаданными.
    -   Конвертация документов.

3.  **Функциональность поиска:**
    -   Проверка корректности и релевантности поисковой выдачи.
    -   Тестирование подсветки результатов.
    -   **Кейс для детальной проработки (на основе недавнего бага):**
        -   **Сценарий:** Поиск со специальными символами и несколькими словами.
        -   **Запросы для теста:** `"договор."`, `"Закон № 218-ФЗ"`, `"копия (1)"`.
        -   **Ожидаемый результат:** Находятся только документы, содержащие все слова из запроса. В результатах отображаются только те фрагменты (предложения), где все эти слова находятся вместе (поиск по близости). Внутри фрагмента все слова из запроса корректно подсвечиваются.

4.  **Управление пользователями (Admin):**
    -   Создание, редактирование, удаление пользователей.

## 3. Инструменты и фреймворки (Предложение)

-   **Unit/Integration:** Jest, React Testing Library (RTL).
-   **E2E:** Playwright или Cypress.
-   **CI:** Все тесты будут автоматически запускаться в GitHub Actions.
